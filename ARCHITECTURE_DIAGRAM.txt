╔═══════════════════════════════════════════════════════════════════════════════╗
║                              DATA FLOW EXAMPLE                                ║
╚═══════════════════════════════════════════════════════════════════════════════╝

User Action: "Thêm bệnh nhân mới"

1. Browser
   │
   └──► GET /new
        │
2. Controller (PatientController)
   │
   ├──► showNewForm()
   │    └──► Forward to patient_form.jsp
   │
   User fills form and submits
   │
   └──► POST /insert
        │
3. Controller
   │
   ├──► buildPatientRequestFromParams()  → Create PatientRequest DTO
   │
   └──► patientService.createPatient(request)
        │
4. Service (PatientService)
   │
   ├──► validatePatientData(request)     → Validate input
   │    ├─ Check username not empty
   │    ├─ Check password not empty
   │    ├─ Check fullname not empty
   │    └─ Check date format
   │
   ├──► Create Patient entity from DTO
   │
   └──► patientDAO.insertPatient(patient)
        │
5. DAO (PatientDAO)
   │
   ├──► DBContext.getConnection()        → Get DB connection
   │
   ├──► PreparedStatement with SQL
   │    INSERT INTO Patients (username, password, ...) VALUES (?, ?, ...)
   │
   ├──► setPatientParameters()           → Set parameters safely
   │
   ├──► executeUpdate()                  → Execute SQL
   │
   └──► Log: "✓ Inserted patient: john_doe"
        │
6. Response
   │
   └──► Redirect to /list                → Show updated list


╔═══════════════════════════════════════════════════════════════════════════════╗
║                           KEY DESIGN PRINCIPLES                               ║
╚═══════════════════════════════════════════════════════════════════════════════╝

✅ Separation of Concerns
   Each layer has ONE responsibility

✅ Single Responsibility Principle
   Each class does ONE thing well

✅ DRY (Don't Repeat Yourself)
   Helper methods, constants, DTO pattern

✅ Dependency Inversion
   High-level modules don't depend on low-level modules

✅ Open/Closed Principle
   Open for extension, closed for modification

✅ Clean Code
   Meaningful names, small methods, proper documentation

✅ Error Handling
   Custom exceptions, proper logging, graceful degradation

✅ Security
   PreparedStatement (SQL injection prevention)
   Input validation
   UTF-8 encoding
